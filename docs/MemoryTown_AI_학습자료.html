MemoryTown AI 학습자료

비전공자도 이해할 수 있게, 이 프로젝트가 무엇을 만들고 어떤 순서로 움직이며 어떤 파일을 보면 되는지 풀어쓴 안내서입니다.

목차 1. 프로젝트 목표 2. 핵심 개념 3. 작동 흐름 4. 파일별 역할 5. 실행 방법 6. AI 모드 7. 과제 조건 매핑 8. 공부 순서 9. 완성 프롬프트

1. 프로젝트 목표

MemoryTown AI는 여러 AI 캐릭터가 하루 일정을 계획하고, 같은 장소에서 만나 대화하고, 대화에서 알게 된 사실을 기억하며, 상대방과의 관계를 점점 업데이트하는 웹앱입니다.

쉽게 말하면 “기억하는 AI 캐릭터 마을”입니다. 그냥 한 번 말하고 끝나는 챗봇이 아니라, 캐릭터가 누구를 만났는지, 무엇을 알게 됐는지, 상대를 어떻게 느끼는지를 저장합니다.

만드는 것

Streamlit으로 만든 웹앱과 제출용 문서, 발표 구성, 테스트, zip 생성 스크립트입니다.

AI가 하는 일

대화 생성, 대화에서 사실 추출, 관계 요약, 하루 장소 계획 생성을 담당합니다.

제출 포인트

Mock, OpenAI, Ollama 모드를 지원하므로 API 키가 없어도 연습 가능하고 실제 AI 실행도 가능합니다.

2. 핵심 개념

Agent

Agent는 이 서비스 안에서 움직이는 AI 캐릭터입니다. 각 Agent는 이름, 나이, 직업, 성격을 가집니다. 예를 들어 “민준, 29세, AI 서비스 기획자, 호기심이 많음” 같은 식입니다.

Memory

Memory는 Agent가 알고 있는 사실 목록입니다. 대화가 끝난 뒤 AI가 “이 대화에서 기억할 만한 사실”을 뽑고, 그 사실을 상대방들의 Memory에 저장합니다.

예: 서연이 “저는 도서관에서 프로젝트를 자주 해요”라고 말하면, 민준의 Memory에 “서연은 도서관에서 프로젝트를 자주 한다”라는 Fact가 저장됩니다.

Relation Map

Relation Map은 “이 Agent가 상대방을 어떻게 기억하고 느끼는지”를 저장하는 관계 요약입니다. 단순한 사실 목록이 아니라 감정과 인상까지 포함한 Reflection입니다.

Reflection

Reflection은 관계 요약 문장입니다. 대화가 쌓이면 “민준은 서연을 차분하고 분석적인 학생으로 기억한다”처럼 상대에 대한 이해가 갱신됩니다.

Daily Plan

Daily Plan은 하루 동안 Agent가 어느 시간에 어느 장소에 있는지 정한 계획표입니다. 이 프로젝트는 08:00부터 20:00까지 2시간 단위로 계획을 세우고, 같은 장소에 있는 Agent끼리 자동 대화를 시킵니다.

3. 앱이 작동하는 흐름

  1. Agent를 준비합니다.
    샘플 Agent 4명이 기본 제공되고, 사용자가 새 Agent를 추가할 수 있습니다.
  2. 대화할 Agent를 고릅니다.
    수동 대화에서는 참여 Agent와 발언 횟수, 선택 주제를 정합니다.
  3. AI가 대화를 만듭니다.
    OpenAI, Ollama, 또는 MockLLM이 Agent 성격과 Memory를 참고해 대화를 생성합니다.
  4. 대화에서 Fact를 추출합니다.
    AI가 각 Agent에 대해 새로 알게 된 사실을 JSON 형태로 정리합니다.
  5. Memory에 저장합니다.
    추출된 Fact는 그 Fact의 본인이 아니라, 대화에 참여한 다른 Agent들의 Memory에 저장됩니다.
  6. Reflection을 업데이트합니다.
    참여한 모든 Agent 쌍에 대해 상대방을 어떻게 느끼고 기억하는지 갱신합니다.
  7. 최종 리포트를 만듭니다.
    특정 Agent 시점에서 상대방별 Fact와 Reflection을 Markdown으로 출력하고 저장합니다.

4. 파일별 역할

파일 역할 비전공자식 설명
app.py Streamlit 화면 사용자가 버튼을 누르고 결과를 보는 웹 화면입니다.
memorytown/models.py 데이터 구조 Agent, Memory, 대화, 계획표 같은 정보의 모양을 정합니다.
memorytown/llm_client.py AI 연결 OpenAI, Ollama, Mock 모드를 같은 방식으로 사용할 수 있게 연결합니다.
memorytown/simulation.py 시뮬레이션 엔진 대화를 실행하고, Fact 저장과 관계 업데이트를 지휘하는 중심 로직입니다.
memorytown/report.py 리포트 생성 특정 Agent 기준으로 알고 있는 사실과 관계 요약을 문서로 만듭니다.
memorytown/storage.py 저장과 불러오기 시뮬레이션 상태를 JSON 파일로 저장하거나 다시 불러옵니다.
data/sample_agents.json 샘플 데이터 처음 앱을 켰을 때 등장하는 기본 Agent 4명의 정보입니다.
tests/ 테스트 대화, Memory 저장, Reflection, 리포트가 제대로 작동하는지 자동 확인합니다.
scripts/make_submission_zip.py 제출 zip 생성 과제 제출에 필요한 파일만 모아 압축 파일을 만듭니다.

5. 실행 방법

먼저 프로젝트 폴더에서 필요한 패키지를 설치합니다.

pip install -r requirements.txt

그다음 Streamlit 앱을 실행합니다.

streamlit run app.py

테스트는 다음 명령어로 실행합니다.

pytest

제출용 zip 파일은 다음 명령어로 만듭니다.

python scripts/make_submission_zip.py

6. AI 모드 이해하기

Mock Mock 모드

API 키나 로컬 AI가 없어도 작동하는 연습 모드입니다. 정해진 규칙으로 대화와 Fact를 만듭니다.

제출 전 실제 AI 시연용으로는 OpenAI 또는 Ollama 모드를 한 번 실행하는 것이 안전합니다.

Cloud Real AI 모드

OpenAI API를 사용합니다. OPENAI_API_KEY와 OPENAI_MODEL을 설정합니다.

OPENAI_API_KEY=실제_API_KEY
OPENAI_MODEL=gpt-4.1-mini

Local Ollama 로컬 모드

내 컴퓨터에서 실행되는 Ollama 모델을 사용합니다. 인터넷 API 키 없이 로컬 AI를 돌릴 수 있습니다.

앱은 Ollama의 /api/tags를 읽어 현재 컴퓨터에 설치된 모델 목록을 자동으로 보여주고, 선택한 모델을 이후 시뮬레이션에 사용합니다.

OLLAMA_HOST=http://localhost:11434
OLLAMA_MODEL=llama3.1:8b

8. 추천 공부 순서

  1. 먼저 앱을 실행합니다.
    화면에서 Agent 생성, 대화 실행, 리포트 생성을 직접 눌러 봅니다.
  2. README를 읽습니다.
    설치, 실행, 제출 방법을 확인합니다.
  3. models.py를 봅니다.
    이 프로젝트가 어떤 정보를 저장하는지 이해합니다.
  4. llm_client.py를 봅니다.
    OpenAI, Ollama, Mock 모드가 같은 기능 이름으로 묶이는 방식을 봅니다.
  5. simulation.py를 봅니다.
    대화 후 Fact 저장과 Reflection 업데이트가 실제로 어디서 일어나는지 확인합니다.
  6. app.py를 봅니다.
    사용자가 보는 버튼과 탭이 내부 기능을 어떻게 호출하는지 연결해 봅니다.
  7. 테스트를 실행합니다.
    pytest가 통과하면 핵심 기능이 깨지지 않았다는 뜻입니다.

9. 한 번에 완성하는 프롬프트

아래 프롬프트는 이 프로젝트를 처음부터 다시 만들고 싶을 때 사용할 수 있는 완성형 요청입니다. 비전공자도 이해할 수 있게 설명, 구현, 테스트, 문서화를 모두 요구하도록 구성했습니다.

너는 Python과 Streamlit을 사용하는 개발 에이전트다. “MemoryTown AI”라는 과제 제출용 프로젝트를 처음부터 끝까지 완성해줘. 목표: AI를 활용한 스타트업 서비스 기획 및 구현 과제를 제출 가능한 상태로 만든다. 서비스 개요: MemoryTown AI는 여러 AI Agent가 가상의 하루 일정을 계획하고, 같은 장소에서 만나 대화하며, 서로에 대한 기억과 관계를 형성하는 AI Agent 관계 시뮬레이션 서비스다. 게임 NPC, 교육용 역할극, 시나리오 제작, AI 캐릭터 테스트 서비스로 확장 가능한 스타트업 아이템처럼 기획한다. 비전공자도 이해할 수 있도록 코드, 문서, README에 쉬운 설명을 포함해줘. 필수 구현: 1. Agent 구조 - Agent는 이름, 나이, 직업, 성격을 가진다. - Agent는 Memory를 가진다. - Memory는 대화에서 추출된 Fact를 라운드별로 저장한다. - Agent는 Relation Map을 가진다. - Relation Map은 다른 Agent와의 관계 요약인 Reflection을 저장한다. - Agent 생성 시 초기 Memory를 주입할 수 있어야 한다. 2. 대화 시스템 - 2명 이상의 Agent가 한 라운드에서 대화할 수 있어야 한다. - 라운드마다 참여 Agent와 발언 횟수 turns를 설정할 수 있어야 한다. - 대화 주제 topic은 선택 사항이다. - topic이 없으면 Agent가 Memory 기반으로 자연스럽게 주제를 꺼낸다. - 처음 만나는 Agent와는 인사와 자기소개부터 시작한다. 3. Fact 추출 및 Reflection - 한 라운드 대화가 끝나면 LLM이 각 Agent에 대한 Fact를 추출한다. - 추출된 Fact는 해당 Agent 본인을 제외한 나머지 참여자의 Memory에 저장한다. - 한 라운드가 끝나면 참여한 모든 Agent 쌍에 대해 Reflection을 업데이트한다. - Reflection은 “이 Agent가 상대를 어떻게 기억하고 느끼는지”를 요약한다. 4. 리포트 - 시뮬레이션 종료 후 특정 Agent 시점의 리포트를 출력한다. - 리포트는 상대방별로 “알고 있는 사실”, “관계 요약 / Reflection” 순서로 보여준다. - 웹 화면에서 볼 수 있고 Markdown 파일로 저장 또는 다운로드할 수 있어야 한다. 5. 자동 시뮬레이션 - 가상의 활동 시간 범위를 08:00~22:00으로 설정한다. - 2시간 단위 시간 슬롯을 사용한다. - 가상의 공간 목록을 정의한다. 예: 집, 학교, 카페, 레스토랑, 도서관, 회사 - 하루 시작 전 각 Agent가 Planning Prompt를 통해 시간대별 장소 계획을 세운다. - 시뮬레이션은 시간 순서대로 진행된다. - 같은 장소에 있는 Agent들끼리 자동으로 대화한다. - 가상의 이틀이 지난 후 특정 Agent 기준 리포트를 파일로 저장할 수 있어야 한다. AI 모드: - OpenAI API를 사용할 수 있게 구현한다. - API Key는 코드에 하드코딩하지 않는다. - .env 또는 환경변수 OPENAI_API_KEY를 사용한다. - OPENAI_MODEL 환경변수로 모델명을 바꿀 수 있게 한다. - Ollama 환경도 인식해 로컬 LLM으로 실행할 수 있게 한다. - OLLAMA_HOST와 OLLAMA_MODEL 환경변수를 지원한다. - Ollama 서버가 감지되면 Ollama 로컬 모드를 선택할 수 있게 한다. - API Key나 Ollama가 없을 때는 MockLLM 모드로 동작한다. - README에 “제출 전 실제 AI 모드 또는 Ollama 로컬 모드로 실행 필요”라고 명확히 적는다. - LLM 결과는 가능하면 JSON으로 파싱한다. - JSON 파싱 실패나 API 오류가 나도 앱이 죽지 않도록 fallback 처리한다. Streamlit 앱: - 모든 UI 텍스트는 한국어로 작성한다. - 사이드바에서 Mock 모드, Real AI 모드, Ollama 로컬 모드를 선택할 수 있게 한다. - 주요 탭은 다음과 같다. 1. 서비스 소개 2. Agent 생성 3. 수동 대화 라운드 4. 하루 / 이틀 자동 시뮬레이션 5. Memory 확인 6. Relation Map 확인 7. 최종 리포트 8. 제출 체크리스트 - 기본 샘플 Agent 4명을 제공한다. - 사용자가 Agent를 추가/수정할 수 있게 한다. - 대화 로그, Fact, Memory, Reflection이 화면에 보이게 한다. - 최종 리포트를 Markdown 파일로 다운로드할 수 있게 한다. 필수 파일 구조: - app.py - memorytown/ - __init__.py - models.py - llm_client.py - simulation.py - report.py - storage.py - data/ - sample_agents.json - docs/ - MemoryTown_AI_기획서.md - MemoryTown_AI_발표자료_구성.md - MemoryTown_AI_학습자료.html - tests/ - test_simulation.py - test_report.py - scripts/ - make_submission_zip.py - requirements.txt - README.md - .env.example - .gitignore 문서: - docs/MemoryTown_AI_기획서.md에는 프로젝트명, 서비스 한 줄 소개, 문제 정의, 목표 사용자, 핵심 기능, AI 활용 방식, 데이터 구조, 구현 플랫폼, 서비스 화면 구성, 스타트업 확장 가능성, 수익 모델 아이디어, 기대 효과, 한계점과 개선 방향, 실행 방법, 과제 요구사항 충족표를 포함한다. - docs/MemoryTown_AI_발표자료_구성.md에는 PPT로 옮기기 쉬운 8~10장 슬라이드 구성을 작성한다. - docs/MemoryTown_AI_학습자료.html에는 비전공자도 이해할 수 있도록 프로젝트 목표, 핵심 개념, 작동 흐름, 파일별 역할, 실행 방법, AI 모드, 과제 조건 매핑, 공부 순서, 완성 프롬프트를 담는다. - README.md에는 프로젝트 소개, 과제 요구사항 매핑표, 설치 방법, 실행 방법, Mock 모드, Real AI 모드, Ollama 로컬 모드, .env 설정 방법, 화면별 사용 방법, 제출 전 체크리스트, 폴더 구조, 테스트 실행 방법, 제출용 zip 생성 방법을 포함한다. 테스트: - MockLLM을 사용해서 테스트한다. - Agent 생성 테스트 - 대화 라운드 후 Memory 저장 테스트 - Reflection 업데이트 테스트 - 최종 리포트 생성 테스트 - 저장/불러오기 테스트 - Ollama 서버가 없을 때 Mock fallback 테스트 제출 zip: - scripts/make_submission_zip.py로 제출용 zip을 만든다. - app.py, memorytown/, data/sample_agents.json, docs/, tests/, scripts/make_submission_zip.py, README.md, requirements.txt, .env.example을 포함한다. - .env, __pycache__, .venv, reports 내부 개인 결과물, .git은 제외한다. 마지막으로: - pytest를 실행해 통과 여부를 확인한다. - streamlit run app.py로 앱이 실행되는지 확인한다. - 제출 zip을 생성한다. - 구현한 내용과 실행 방법을 짧게 요약해줘.
